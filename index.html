Projects
Faction Wars

Background

In 2016, a group of ten others and I decided to work on a massive scale game mode for Garry's Mod. After hours of deliberation, we came up with Faction Wars. The game mode premise is that of a combat-oriented roleplay game mode, where players join "factions". Players collaborate with their factions to launch wars, capture property, and construct illegal enterprises.

Features

Below are some of the major features of the game mode

  • Fully custom UI
  • 42 custom entities such as tools, weapons, turrets, radars, and equipment/minerals (used for enterprise system)
  • Advanced resource system (includes crafting, etc.)
  • Factions (teams with the ability to share funds, capture land and more)
  • Zones (areas which can be captured by factions; can be customized by server admins for location and capturability)
  • In-game voting system

Contributions

This project was completed over seven months from June to December of 2016. During this project, I contributed more than 16,000 lines of code with over 180 commits. The scope of my work included all the major features listed above, plus things such as in-game chat commands and more.

Challenges

One of the major challenges was the team size. At eleven total contributors, the scale of the team for this project was somewhat large. It was difficult to manage who was doing what and to keep code consistent. We mitigated this risk by staying in constant communication on what we were doing. As well, we instituted a code style guide early and used a Trello board to keep in-depth descriptions of what we were doing and the status. Further, we would commit changes early and often to prevent conflicts and repeated work.

Another major challenge (for me) was the experience I had going into this project. I had never worked on a large scale project like this with others before, so I had to quickly learn how to do this. In addition to teamwork, my programming skills faced massive challenges. Features such as the turret, radar, voting system, and others, were out of my experience level. I had to continually research and learn to accomplish each task. By trial and error, I was able to create the fully functioning and intuitive features found above.

Player Menus and HUD

This video showcases each of the in-game menus including the Players (where you can view other players), Jobs (where you can switch factions and roles), Items (where you can purchase resources, machines, and weapons), and the Faction menu (where you can manage the faction bank, and view other players in your faction). As well, you can view each aspect of the player's HUD, including health, money, faction, location, and notifications.

Resources and Machinery

In this video, we see how the resource system works. In this example, the player is trying to farm opioids to sell. To do that, the player needs an Opioid Refinery, which requires power, raw resources, and water. Water is supplied through a container. Raw resources are mined through a Raw Resource Extractor, which requires power. Both machines are powered by a generator, which requires gas. From this, we can see how each system is linked and each subset is derived from another.

Zones / Capturing

In this video, we see how the player can view and capture zones. By holding the alt key and looking down, the player can see an auto-generated map of preconfigured zones and who controls them. By using this map, players can find locations to capture. Once they enter the location, the majority faction begins to take it over, as is seen on the map.

AdminMe

Background

AdminMe was a personal project started in early 2017. The goal was to create a comprehensive administration system for Garry's Mod utilizing in-game and web platforms. While playing, administrators can manage users, bans and more with over 50 commands and an intuitive UI. The web system allows users to login via Steam and to customize aspects such as ranks, users, and more.

Features

The in-game platform features the following

  • Fully custom and intuitive UI
    • UI allows you to execute commands, view warnings/player events/logs, manage bans, and manage music
  • Over 50 commands to perform actions such as banning/kicking players, teleporting, reporting
    • Each command can be run from chat, the developer console, or the UI
The web platform features the following
  • Intuitive UI via LuminoAdmin panel
  • Login integration via SteamAuth, which verifies you also are the correct rank to have access
  • Ability to manage users, ranks, and permissions, view logs/warnings, and manage bans. As well, publically, the site has a User List and Ban List, which anyone may see
Additional planned features include
  • Revised timed rank integration
  • More visually appealing in-game menu
  • Direct server communication from the web platform via sockets
    • This would allow for server administration without being in-game, as well as the ability to view server output directly

Contributions

This project was started in 2017 and has been improved upon since. I am currently working on the third revision (started in early 2018). I have contributed most of the code, besides the UI for the web platform, teleportation commands, and in-game command parser. Total, this leads to around 4000 lines of code for the in-game system.

Challenges

One of the major challenges was learning how to manage a project of this scope by myself. Because this was an individual project, I had no idea where to begin and how to move forward. I began by referencing other administration add-ons and then compiling key features I wanted on Trello. By staying organized and detailing ideas, I was able to begin developing the base and then branched out to other key features.
